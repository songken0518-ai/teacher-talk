🔑 參考答案（教師版｜學生版不顯示，供你對照）
  1. 放在自己電腦時，它只在你的區域網路裡、用本機或私有位址（file:// 或 127.0.0.1），沒有對外的公有 IP，外面找不到它；一放上網路（例如 GitHub Pages）後，它住在一台有公有 IP、又有好記網域名稱的伺服器上，透過網際網路，全世界就都連得到。
  2. 大致五關：① 你打的網域名稱先經 DNS 翻成 IP → ② 你的要求被切成一個個封包、貼上地址和編號 → ③ 經路由器、走網際網路一站站接力送到伺服器 → ④ 伺服器一問一答把資料（最好上了 https 的鎖）回傳 → ⑤ 你的裝置照編號重組成螢幕上的畫面。
  3. 只要會送出重要或私密資料的時候——登入打帳號密碼、線上付款、填個資表單——就一定要先確認網址是 https、有 🔒 鎖頭（代表傳輸有加密），在公共 Wi-Fi 上尤其要看。提醒：鎖頭只保證「傳輸加密」，不代表網站是好人——釣魚網站也可能有 https，所以還要認清楚網址本身。

01:00區域網路 vs 網際網路第 1 章家裡的小網（LAN）只在本地相通；要連外面得走網際網路。
  • 兩個定義：區域網路＝同一個地方（家、教室）裝置互連的小網；網際網路＝把全世界無數小網串起來的大網。
  • 學生有感的例子：AirDrop、教室區網印表機、多人區網遊戲——不用外網也能互傳，那就是 LAN。
  • 為什麼要分兩層：LAN 又快又免費，但要找「外面的人」就得經過對外門口、上網際網路。
  • 先埋 4 章的鉤子：「連上 Wi-Fi＝連上網際網路」是錯的——連上 Wi-Fi 只是進了 LAN。
  • 🎬 畫兩個圈：小圈 LAN、大圈 Internet，中間一道門＝路由器。
02:30資料切成封包旅行第 1 章要求被切成小封包、編號、接力送達、重組；掉包會重傳。
  • 核心原理：資料不是整包送，是切成很多小封包，每個都貼上「來源、目的地、編號」。
  • 為什麼要切小：大家共用線路，小封包能穿插、塞車可繞路、壞一個只要補一個。
  • 比喻：把一本書拆成一頁頁、每頁寫頁碼分開寄，對方照頁碼裝訂回原書。
  • TCP 點名重傳：收到後檢查編號，缺哪個就請對方只重傳那一個，保證完整。
  • 生活例子：影片轉圈緩衝、傳大檔變慢，就是封包在路上塞車或重傳。
  • 🎬 可現場跑頁面的「封包切割」小動畫；下一格接「送到哪個地址」。
04:00IP 門牌與 DNS 翻譯員第 2 章IP 像門牌但難記，改用網域名稱，靠 DNS 翻成 IP。
  • IP＝門牌號：每台上網的機器都有一串數字位址，電腦只認數字。
  • 問題與解法：數字難記，所以用好記的網域名稱（google.com）；DNS 就是網路的電話簿/翻譯員。
  • 完整流程：你打名字 → 問 DNS → 拿到對應 IP → 封包才知道要送去哪。
  • 比喻：你記朋友「名字」不記手機號，通訊錄幫你對應——DNS 就是網路的通訊錄。
  • 破迷思：DNS 不是防毒、不是防火牆，它只做「名字 → 號碼」的翻譯。
  • 🎬 可玩頁面 DNS 查詢小工具，點名字→跳出 IP。
05:00有沒有上鎖：http vs https第 3 章https 比 http 多一把鎖、會加密；登入付款要看鎖頭。
  • 差別：http＝沒上鎖，資料在路上是明文，被攔就看光光；https＝有加密，被攔也是一堆亂碼。
  • 怎麼看：網址列的 🔒 鎖頭、開頭是 https。
  • 比喻：http 像寄明信片（郵差都看得到）；https 像上鎖的鐵盒。
  • 規則：登入、打密碼、刷卡前一定要有鎖頭；用公共 Wi-Fi 時更要盯。
  • 破迷思（連第 5 章）：有鎖頭只保證「傳輸」加密，不代表網站是好人——釣魚網站也能有鎖頭。
  • 🎬 玩頁面加密示範：切換上鎖/沒鎖，看攔截到的內容差別。
06:00上網是一問一答：主從架構第 3 章瀏覽器負責問、伺服器負責答。
  • 定義：客戶端（你的瀏覽器）負責發問／要資料，伺服器負責回答／給資料。
  • 一問一答：你點一個連結＝送出一個請求，伺服器回一個回應（網頁、圖片…）。
  • 比喻：餐廳點餐——你（客戶端）點、廚房（伺服器）出餐；你不會自己衝進廚房煮。
  • 生活例子：YouTube 你點影片（問），它把影片串流回來（答）。
  • 為什麼重要：整個網路幾乎都是這種「請求—回應」；第 8 章雲端、第 9 章 AI 都是它的放大版。
  • 🎬 可玩頁面「請求/回應」對話動畫。
07:00背後默默工作的裝置第 4 章路由器、Port、Wi-Fi、基地台讓資料真的送得出去。
  • 路由器：發 Wi-Fi、發私有 IP、當家裡對外唯一門口；用 NAT 讓全家共用一個公有 IP。
  • Port（房間號）：一台機器同時跑很多服務，用 Port 區分（80 http、443 https、53 DNS）。
  • Wi-Fi：用電波連路由器；2.4GHz 傳遠穿牆、5GHz 快但短。
  • 基地台：在外沒 Wi-Fi 靠它，移動時訊號換手不斷線（4G/5G）。
  • 破迷思：連上 Wi-Fi ≠ 能上網——可能卡在「路由器到外網」那段。
  • 🎬 玩頁面 NAT 示範：一直加裝置、看對外都共用一個公有 IP。
08:00保護你自己：資訊安全第 5 章防釣魚、惡意軟體、弱密碼；強密碼＋兩步驟＋認網址。
  • 三大威脅：釣魚（假網站騙帳密）、惡意軟體（亂裝亂點中招）、弱密碼（好猜或到處共用）。
  • 強密碼關鍵：長度比複雜更重要——用一句只有你懂的長句最好記又難破。
  • 不重複：一個網站被盜，其他不會跟著淪陷。
  • 兩步驟驗證：密碼被偷還有第二道關（手機驗證碼）。
  • 認網址：看鎖頭、看網域有沒有偷改字（g00gle、paypa1）。
  • 生活例子：假中獎簡訊、假物流通知連結。🎬 玩釣魚辨識遊戲搶答。
09:00自己動手做、發佈給全世界第 6 章＋實作作業用 VS Code 寫前端、Live Server 預覽、GitHub 發佈。
  • VS Code：專業程式編輯器，有自動補全、即時預覽，比記事本好用太多。
  • 三分工：HTML＝骨架（內容）、CSS＝外觀（樣式）、JS＝互動（行為）。
  • Live Server：存檔就自動更新畫面——其實是本機開了一個小伺服器（回收主從架構）。
  • GitHub Pages：把網頁放上網際網路，變成全世界都連得到的網址（回收開場謎題）。
  • 收尾金句：整門課從「別人連不到」走到「全世界都連得到」——這趟旅程你自己走完了。
  • 🎬 預告期末作業：用 AI 生成部落格、發佈到 GitHub。

✅ 課堂口頭抽問（看完影片問這三題）＋參考答案
Q1為什麼網頁放自己電腦別人連不到，一放上網路就全世界打得開？
參考答案：差別不是「網頁本身」，而是有沒有放到一台「全世界都找得到、而且對外開放」的伺服器上。
  • 放自己電腦時，網頁只存在你這台機器的區域裡，沒有對外的公開地址，別人不知道怎麼連、也沒有路走得到你這台。
  • 一放上網路（例如 GitHub、某個伺服器），它就有了對外的 IP＋網域名稱並對外開放；任何人打那個網址 →（DNS 找到 IP）→ 封包一路接力送到那台伺服器 → 網頁再傳回來，就打得開。
  • 🎬 可延伸：這就是期末作業用 GitHub Pages 發佈的意義——把本機網頁放上網際網路，變成全世界都連得到的網址（回收開場謎題）。
Q2你打的一個網址，從「名字」變成螢幕上的「畫面」，中間大概經過哪些關卡？
參考答案：大致經過這幾關（能講出 DNS、封包、路由、伺服器、重組就很好）：
  • ① DNS 翻譯：把網域名稱（名字）翻成 IP（門牌號）。
  • ② 切封包：你的要求被切成小封包，各自貼上來源／目的地／編號。
  • ③ 路由接力：封包經路由器、網際網路一站一站送到目標伺服器。
  • ④（https）加密：若是 https，傳輸中的資料是加密的。
  • ⑤ 主從一問一答：瀏覽器發請求（問）、伺服器回應（答），把網頁內容送回。
  • ⑥ 重組＋渲染：你的裝置照編號把封包重組，瀏覽器畫成畫面。
一句話：名字 →(DNS)→ IP → 切封包 → 路由接力 → 伺服器一問一答 → 傳回重組 → 畫面。
Q3什麼時候你一定要看到網址上的「🔒 鎖頭」才安全？
參考答案：只要你要輸入敏感資料時，就一定要看到鎖頭（https）：
  • 登入帳號密碼、刷卡／付款、填個資（身分證、地址、電話）、網路銀行、輸入驗證碼等。
  • 為什麼：鎖頭代表 https、傳輸加密，被攔到也是亂碼；沒鎖頭（http）＝明文寄明信片，帳密可能被看光。
  • 特別提醒：用公共 Wi-Fi 時更要看鎖頭（同一網路別人較容易偷看）。
  • 🎬 破迷思：有鎖頭只保證「傳輸加密」，不代表網站是好人——還要看網域有沒有被偷改字（g00gle、paypa1，第 5 章釣魚）。所以「有鎖頭」是必要條件、不是萬能保證。

區域網路與網際網路，資料切成封包旅行

區域網路 / 網際網路 ‧ 封包 ‧ 路由 ‧ TCP/IP
1

網路依照涵蓋的範圍，可以分成兩種，先認識它們的差別。一種叫「區域網路」(LAN)：把一個地方的機器連起來，例如你家的手機、電視、筆電，靠同一台路由器互通；學校電腦教室也是一個。它是個「自己人圈圈」，圈內互相找得到，出了圈外面看不到（像你用 AirDrop 把照片傳給旁邊同學，根本沒出門，這就是區域網路）。另一種叫「網際網路」：把全世界一個一個區域網路串成一張超大的網（像你用 LINE 把訊息傳給遠在日本的朋友，就是走進了網際網路）。所以一份網頁只放在一台電腦裡，別人連不到；要全世界連到，得先走進網際網路。

資料送到對方那裡的方式是這樣：不是整包送，而是先切成一個個「封包」。每個封包都很小，上面寫三件事——要送到哪（目的地）、從哪來（來源）、自己是第幾號（編號），方便對方照號碼組回去。之所以要切，是因為切小塊才好塞縫隙、走不同的路，而且掉了一塊只要補那一塊，不用整個重傳。

封包送出去後，會經過一台又一台路由器，一站一站接力轉送，每台看目的地決定丟給下一台，像郵件經過轉運站。神奇的是：同一筆資料的不同封包，可能走不同的路，哪條順走哪條，最後在終點會合。

封包離開你家路由器後，先到電信公司，也就是 ISP，再上更大條的骨幹網路跨城市、跨國；而國與國之間，大多靠鋪在海底的光纖電纜傳送。所以你傳給國外朋友的封包，常常真的「潛到海底」再上岸。

最後，靠 TCP/IP 這組規矩確保不漏不亂：IP 負責把封包送到對的地方，TCP 負責切割、編號、檢查有沒有到齊，少了哪一塊就叫對方重傳。封包到了對方伺服器，伺服器把回應也切成封包送回，你的電腦再照編號重新組合，瀏覽器畫成你看到的畫面——整趟來回通常零點幾秒。

🌍 生活中的例子
你可能聽過「頻寬」和「延遲」。頻寬像水管多粗——管越粗，同一時間能流過的水（資料）越多，下載大檔就越快；延遲像水從一頭流到另一頭要多久——距離越遠（例如連到國外伺服器），來回時間越長。打國外的線上遊戲會「卡卡的」，多半是延遲太高，不一定是頻寬不夠——延遲其實就是你打遊戲常說的 ping 值（單位毫秒，數字越小越順）。
🎤 第 9 塊 · 老師照著念

講兩個你一定聽過、卻常搞混的詞：頻寬和延遲。頻寬像水管多粗——管越粗、同一時間流過的資料越多，下載大檔越快；你家方案寫的『100M、300M』就是頻寬。延遲像水從一頭到另一頭要多久，就是你打遊戲講的 ping 值，單位毫秒，連台灣十幾毫秒、連美國一百多毫秒。

重點來了，這是最多人誤會、課本沒點破的：打國外遊戲卡，多半是延遲太高、不是頻寬不夠。所以記住——下載慢怪頻寬、遊戲卡怪延遲，有時你網速很快、遊戲還是卡，就是距離遠、延遲高。（現場 ping 一個國內站、一個國外站，把 ms 寫黑板對比）

找到對方：IP 門牌 + DNS 名字

私有/公有 IP ‧ 網域名稱 ‧ DNS
1

每台上網的機器都有一個 IP 位址，像門牌（192.168.1.100）。私有 IP（192.168 或 10. 開頭）只在自己的區域網路裡有效、會重複；公有 IP 才是全世界唯一、查得到的正式地址。所以自己電腦的私有 IP 不能給全世界看，外面找不到。

我們把一個 IP 拆開看：它由四段數字組成，中間用點隔開（像 192.168.1.100）。每一段都只能是 0 到 255 之間的數字，不會出現 300、999 這種。至於為什麼是 0~255，是因為電腦其實是用「8 個 0 與 1」來記每一段，而 8 個位元最多剛好能表示 256 種狀態（0~255）。四段合起來，全世界大約能編出 43 億 個位址——聽起來很多，但現在手機、電腦、電視、手錶、感測器全都要上網，早就不夠用了，所以又設計了更長、數量幾乎用不完的新版位址叫 IPv6（你偶爾會看到一長串像 2001:0db8:… 的位址，就是它）。我們課堂上看到的四段式，是舊版的 IPv4。

家裡每台裝置的私有 IP，是由路由器決定的。每當有新裝置連上 Wi-Fi，路由器就會自動發一個還沒人用的私有 IP 給它（這個自動分配的機制叫 DHCP），所以你連 Wi-Fi 從來不用自己手動打一串數字，開機就能上網。也因為是「自動分配」，你今天拿到的 IP 和明天可能不一樣，這很正常。

IP 是一串數字很難記，所以有好記的網域名稱（像 google.com）。電腦看不懂名字、只懂數字，中間靠一位翻譯員 DNS，你一打名字，DNS 就幫你查出對應的 IP，電腦才知道連去哪。

DNS 查名字其實是一層一層問的：你的電腦會先去問離你最近的一台 DNS 伺服器（通常是電信公司或路由器提供的）。如果它剛好記得（之前有人查過、暫時存起來，叫「快取」），就立刻回答你，所以你常逛的網站開得特別快；如果它不知道，才會往上去問管理該網域的伺服器，查到之後也順便記下來，下次就快了。整段一問一答通常只要零點幾秒，快到你幾乎沒感覺。

主題 03

安全連線：上鎖、一問一答、驗身分

HTTPS ‧ 主從架構 ‧ 身分驗證
1

網址前面的 http / https 是溝通規矩。https 比 http 多一把鎖（加密），半路被攔也只是亂碼；登入、付款一定要看網址列的鎖頭。

「加密」的意思很簡單，就是把你要傳的內容用一套只有你和對方看得懂的方式打亂。中間任何人（例如同一個 Wi-Fi 上的陌生人、沿路經過的設備）就算把資料整包攔下來，看到的也只是一堆亂碼，拼不回原文。http 沒有這把鎖，你打的帳號密碼等於用「明信片」寄出去，誰經手都看得到；https 上了鎖，就像裝進密封信封，只有收件的伺服器拆得開。

瀏覽器要分辨對方是「真的」那家網站、還是假冒的，靠的是：正牌網站手上有一張由公正第三方核發的「憑證」，像一張蓋了official章的身分證。瀏覽器連上去時會先驗這張憑證，對得上、又有 https，才會顯示鎖頭；對不上就會跳出「不安全」的警告。所以那個小鎖頭，其實同時代表了「有加密」和「身分驗證過」兩件事。

再看上網的本質：它是一問一答。你的瀏覽器（客戶端）負責「問」，對方的電腦（伺服器）負責「答」。你打開一個網站，背後其實是瀏覽器對伺服器說「請把首頁給我」，伺服器回「好，網頁在這」；網頁上每一張圖片、每一段影片，也都是這樣一筆一筆要回來的。這種「一邊負責問、一邊負責答」的設計，就叫主從式架構。

按「用 Google 登入」時，Google 認的是網址門牌有沒有登記、有沒有上鎖：在名單上又有 https 就放行，看不懂的私有 IP、沒登記又沒上鎖就擋下。這是在保護你——假網站網址對不上，就拿不到你的帳號。

🎤 第 1 塊 · 老師照著念

這堂講『安全連線』，簡單說就是怎麼上網才不會被偷看、被騙，跟你的個人安全很有關係。先看網址前面的 http 跟 https，它們是電腦溝通的規矩。重點是：https 比 http 多了一把鎖、也就是加密，資料加密後就算半路被攔，看到的也只是亂碼、偷了沒用。所以記住保命的一句：登入、付款，網址列一定要先看到那個鎖頭。

那『加密』到底是什麼？就是把你要傳的內容用一套只有你和對方看得懂的方式打亂。中間任何人——同一個 Wi-Fi 上的陌生人、沿路的設備——就算整包攔下來，看到的也只是亂碼、拼不回原文。（可用『每個字往後移一格』的暗號現場示範加密、再解回來，讓學生看到被攔也看不懂）所以可以這樣比：http 像寄明信片，帳密誰經手都看得到；https 像裝進密封信封，只有收件的伺服器拆得開。

路由器是家裡網路的核心，它一台同時做好幾件事：對內發出 Wi-Fi 訊號、自動發私有 IP 給每台裝置（前一章說的 DHCP）；對外當整個家連到網際網路的唯一門口。問題來了：電信公司通常只給你家一個公有 IP，但家裡有手機、筆電、電視一大堆裝置，要讓大家一起上網，靠的是 NAT 技術——路由器幫每台裝置「換地址」：對內它們各有不同的私有 IP，對外全部共用那一個公有 IP，回來的資料再由路由器分送回正確的裝置。所以全家上網不會打架，外面也只看到一個地址。

找到一台機器靠 IP，但一台機器上可能同時跑很多種服務（網站、收信、傳檔…），要分清楚資料是要給哪個服務，靠的是 Port（連接埠）。如果說 IP 是一棟大樓的地址，Port 就是裡面的房間號：資料用「IP + Port」一起標，才能準確送到對的服務。有些 Port 是大家約定好的「常用房號」，例如 80 給 http、443 給 https、53 給 DNS，所以你打網址時不用自己指定號碼，瀏覽器會自動用對的 Port。

Wi-Fi 是用電波連上路由器的，常見有兩個頻段：2.4GHz 傳得遠、穿牆強，但速度較慢、容易被其他電器干擾；5GHz 速度快，但傳不遠、怕牆擋。所以離路由器近就用 5GHz、隔了好幾道牆就用 2.4GHz。要特別記住一件常被搞混的事：「連上 Wi-Fi」不等於「能上網」——連上 Wi-Fi 只代表你進到那個路由器的區域網路，如果路由器本身沒接到外面（例如電信線路斷了），你還是連不到網際網路。

手機在外面沒有 Wi-Fi 時，靠的是基地台（就是路邊或屋頂那些電信天線）。手機發電波給最近的基地台 → 基地台接上電信公司的網路 → 再上網際網路。神奇的是你邊走邊講電話或看影片也不會斷：當你離開一座基地台的範圍，訊號會自動交棒給下一座（這個動作叫換手 Handover），整個過程你完全感覺不到。我們常聽到的 4G、5G，就是行動網路一代比一代更快的技術。

這堂看網路背後的裝置，先講最核心、你家一定有的——路由器。它一台同時做好幾件事：對內發 Wi-Fi 訊號、用上一章說的 DHCP 自動發私有 IP 給每台裝置；對外，當整個家連到網際網路的唯一門口。順帶釐清一個超多人搞混的：『數據機 modem』和『路由器 router』不一樣，但家用常兩個合成一台，就是電信裝的那個一排燈在閃的盒子。

接著是上一堂留的鉤子：電信公司通常只給你家一個公有 IP，但家裡手機、筆電、電視一大堆，要讓大家一起上網，靠 NAT 技術——路由器幫每台『換地址』：對內各給不同私有 IP、對外全部共用那一個公有 IP，回來的資料再由路由器分送回正確的裝置，所以全家上網不會打架、外面也只看到一個地址。

那一台機器上同時跑很多服務（網站、收信、傳檔），怎麼分清資料要給誰？靠 Port（連接埠）。IP 是一棟大樓的地址，Port 就是裡面的房間號，資料用「IP＋Port」一起標才能準確送到。有些是大家約定好的常用房號：80 給 http、443 給 https、53 給 DNS，所以你打網址不用自己指定，瀏覽器會自動用對的 Port。

再講 Wi-Fi：它是用電波連上路由器的，常見兩個頻段——2.4GHz 傳得遠、穿牆強，但速度較慢、容易被干擾；5GHz 速度快，但傳不遠、怕牆擋。所以離路由器近用 5GHz、隔好幾道牆用 2.4GHz。這裡要特別記一件最多人搞混的事：『連上 Wi-Fi』不等於『能上網』——連上只代表你進了這台路由器的區域網路，如果路由器本身沒接到外面（電信線路斷了），你還是上不了網。

最後，手機在外面沒 Wi-Fi 時，靠的是基地台（路邊、屋頂那些電信天線）：手機發電波給最近的基地台 → 基地台接上電信公司的網路 → 再上網際網路。神奇的是你邊走邊講電話或看影片也不會斷：一離開某座基地台的範圍，訊號會自動交棒給下一座，這個動作叫換手 Handover，你完全感覺不到。我們常聽的 4G、5G，就是行動網路一代比一代更快的技術。（黑板畫：路由器左邊接一堆裝置、各自不同私有 IP，右邊一條線出去、共用一個公有 IP，NAT 寫中間；再畫手機在兩座基地台間「交棒」）

解答：因為大家透過同一台路由器對外，NAT 讓你們共用一個公有 IP；對內路由器發給每人不同的私有 IP 來區分。

  • 路由器：發 Wi-Fi、用 DHCP 發私有 IP、當對外門口；用 NAT 讓全家共用一個公有 IP。

資訊安全：保護你的資料和帳號

威脅 ‧ 自保三招
1

資安就是保護你的資料和帳號，不被偷、不被騙、不被破壞。先認識四種最常見的威脅：

一、釣魚（Phishing）：壞人做一個長得幾乎一模一樣的假網站或假訊息，騙你輸入帳號密碼。它常假裝成「你的帳號異常，請立刻登入」「包裹無法配送，請點連結」，用製造緊張讓你來不及細看。破綻通常藏在網址裡（拼錯一個字、多一個怪字、網域不對）。

二、惡意軟體（Malware）：偽裝成遊戲外掛、破解程式、來路不明的附件，裝進你電腦後偷資料或搞破壞。所以別亂裝來路不明的程式，系統和防毒也要定期更新（更新常常就是在補漏洞）。

三、中間人攻擊：壞人躲在你和網站中間偷看、甚至竄改你傳的內容，最容易發生在沒加密的 http 連線和不安全的公共 Wi-Fi。對策就是前面學過的——認 https、看鎖頭，加密後他偷到的只是亂碼。

四、弱密碼 / 重複密碼：密碼太簡單（像 123456、生日）很快被猜中；更危險的是每個網站都用同一組密碼——只要其中一個網站外洩，壞人就拿這組去試你所有的帳號，一個破、全部遭殃。

知道威脅後，自保其實有三招最有效：第一，密碼要夠長、夠難猜，而且每個重要網站都不一樣（記不住可以用瀏覽器或密碼管理工具幫你記）；第二，開啟「兩步驟驗證」——登入時除了密碼，還要再輸入手機收到的一次性驗證碼，這樣就算密碼被偷，壞人沒有你的手機也進不去；第三，登入或付款前先認網址、看鎖頭。這三招裡，光是開兩步驟驗證，就能擋掉絕大多數的帳號盜用。

看這張對照表。比較安全的：夠長（12 字以上）、大小寫數字符號混合、每個重要網站都不一樣、用一句只有你懂的話改編。容易被破的：太短、只有單純數字或英文、到處同一組、用生日姓名 123456。

這裡有個最反常識、最該記的：長度往往比複雜度更重要，一個又長的密碼比又短又怪的難破非常多倍，因為長度一加、要試的組合爆炸性變多。所以教你一招最實用的：用一句只有你懂的話改編，又長又好記又難猜，等下密碼工具會讓你親眼驗證。

🌍 生活中的例子
收到「你的帳號將被停用，請立即點此登入」的簡訊或 Email，先別急著點。正規公司不會用這種急迫語氣逼你交帳密——這是最典型的釣魚。要登入就自己打官方網址進去，不要點訊息裡的連結。
🎤 第 7 塊 · 老師照著念

舉個你一定收過的：『你的帳號將被停用、請立即點此登入』——先別急著點。正規公司不會用這種急迫語氣逼你交帳密，這就是最典型的釣魚。幫你提煉成三秒辨識法，記三個特徵：一、假裝官方；二、製造緊迫（限時、停用、罰款）；三、要你點連結登入。

正確做法記死一句：不要點訊息裡的任何連結，要登入就自己開官方 App、或自己手動輸入官網。口訣——自己走正門、不走別人遞來的後門。

想一想：為什麼「公共 Wi-Fi + http 網站」特別危險？
解答：因為公共 Wi-Fi 上可能有壞人，而 http 沒有加密，你送的帳密是「明信片」狀態，同網路的人就能偷看。改用 https 或行動網路就安全多了。
🎤 第 9 塊 · 老師照著念

想一想，這題要把兩堂課串起來：為什麼『公共 Wi-Fi 加 http 網站』特別危險？（停一下，引導學生把『Wi-Fi 有壞人』和『http 沒加密』拼起來）

答案：這是兩個危險疊加——公共 Wi-Fi 上可能有壞人，加上 http 沒加密、密碼是『明信片』狀態，等於在有小偷的房間裡把密碼寫明信片上寄出去。怎麼辦？改 https 或改行動網路就安全多了。

先用五分鐘建立大圖像：前面五章都在講『網路』那些看不見的觀念，這一堂換個味道——你要親手做出看得見的東西，也就是前端。先把『前端是什麼、用什麼工具』講清楚，再開始動手。

2

前面一直在講網路觀念，現在要動手寫一點前端——也就是瀏覽器裡看得到的畫面（文字、按鈕、顏色、圖片）。做一個網頁其實是三種語言一起合作：HTML 負責內容與骨架（頁面上有哪些文字、按鈕）、CSS 負責樣式（顏色、大小、排版好不好看）、JavaScript（JS）負責互動（按下去會發生什麼事）。

我們用的工具是 VS Code（Visual Studio Code），它就像專門寫程式用的記事本，有顏色標示、自動排版，很好上手。學校的電腦都已經安裝好 VS Code 了，打開就能直接用，不用再下載。

🎤 第 2 塊 · 老師照著念

今天動手寫的是前端，也就是瀏覽器裡看得到的畫面。做網頁是三種語言合作，這三個一定要先記住分工：HTML 管內容和骨架、CSS 管樣式好不好看、JavaScript 管互動（按了會發生什麼）。工具用 VS Code，像寫程式專用的記事本，學校電腦都裝好了，打開就能用。

✅ 怎麼用 Live Server 預覽

先問你一個每天都遇到的怪事：你昨天只是上網查了一雙球鞋，今天打開臉書、YouTube，怎麼到處都是那雙鞋的廣告，好像被跟蹤了？這一章就來拆穿它——關鍵是一個叫 Cookie 的小東西。

Cookie 是網站存在你瀏覽器裡的一張「小紙條」。你第一次逛某個網站，它就在你電腦留一張紙條，寫著「這個人是幾號、剛剛看了什麼、購物車放了什麼」；下次你再來，網站讀這張紙條就認得你了。所以 Cookie 其實很有用——它讓你登入一次就不用一直重打密碼、購物車東西不會不見、網站記得你的語言設定。

但 Cookie 分兩種，差別很重要。第一方 Cookie：你正在逛的那個網站自己留的紙條，多半是為了服務你（記住登入、購物車），這種還好。第三方 Cookie：頁面上別家廣告公司偷偷夾帶的紙條——你在 A 網站看了鞋、在 B 網站又被同一家廣告商認出來，它就把你跨網站的足跡串起來，於是鞋廣告一路跟著你。這就是「被廣告跟蹤」的真相：不是網站在偷看你，是同一家廣告商在很多網站都放了紙條。

那「無痕模式」能防嗎？這裡有個天大的誤會要講清楚：無痕只是「不在你這台電腦留紀錄」——關掉視窗後，這台電腦查不到你去過哪。但是！你連過的網站、學校或公司的網路管理者、你的電信公司（ISP），全都還是看得到你去了哪裡。無痕不是隱形斗篷，它防的是「共用電腦的家人同學」，不是「網站和網路管理者」。

所以怎麼保護自己的隱私？記三招：① 那些「本網站使用 Cookie，同意嗎」的視窗，不用全部按同意——選「只允許必要」就好，拒絕的是拿來追蹤你的第三方 Cookie；② 用完公共電腦一定要登出、順手清掉 Cookie；③ 別在來路不明的網站亂登入、亂授權。你的瀏覽足跡就是一種個資，值得保護。

雲端是什麼：其實就是「別人的電腦」

雲端 ‧ 資料中心 ‧ 好處與風險 ‧ 主從架構大規模版
1

你一定常聽到「把檔案存到雲端」「照片自動上傳雲端」。先破一個最大的誤會：雲端不是天上的雲，也不是飄在空中的魔法。雲端說穿了，就是「別人的電腦」——是某家公司（Google、Apple、微軟…）放在資料中心裡、一整排一整排、24 小時開著的大電腦。你把檔案「存到雲端」，其實是透過網際網路，把檔案送去存在他們的電腦上。

其實你每天都在用雲端，只是沒發現：Google 相簿、iCloud、Google 雲端硬碟幫你存照片檔案；YouTube、Netflix、Spotify把影片音樂放在他們的電腦、串流給你看；連很多遊戲的存檔也在雲端，所以你換手機、進度還在。這一切能成立，靠的正是前面幾章——資料切成封包、走網際網路、用 https 安全地送到對方伺服器再傳回來。雲端就是「主從架構」的大規模版：你的裝置負責問，雲端的電腦負責存與答。

那為什麼大家愛用雲端？四個好處：① 不佔你的空間——幾萬張照片存雲端，手機不會爆；② 換裝置也在——手機掉了、換新機，登入帳號檔案照樣回來；③ 多人可共用——一份 Google 文件，全組可以同時打開一起編輯；④ 自動備份——你不用手動存，它幫你留好幾份，不怕單一硬碟壞掉。

但雲端也有代價，要清楚：① 斷網就打不開——東西在別人電腦上，沒網路就拿不到（除非事先設定離線）；② 隱私交給別人——你的照片、文件放在別家公司手上，等於信任它會保管好、不亂看；③ 帳號被盜就全都露——所有東西集中在一個帳號，密碼一旦外洩，全部一起遭殃。所以第 5 章的資安——強密碼、不重複、開兩步驟驗證——在雲端時代更重要，因為你把命脈都放上去了。

最後給個簡單分類，聽過就好：雲端服務大致有三種——有的租給你放檔案／算力（像雲端硬碟、租一台雲端主機），有的直接給你現成的服務用（像 Google 文件、Gmail、你要用的各種網站）。你不必自己買一堆設備，用多少、租多少，這就是雲端最迷人的地方。

✏️ 課後練習

1. 你在手機上用 AI，為什麼說「用 AI 其實就是在上網」？
因為 AI 模型太大、跑在雲端的大電腦上；你的問題透過網際網路送出、算完再把答案傳回，過程和上網站一樣走封包、https、主從架構。
2. 用 AI 時，關於「隱私」最重要的一條原則是什麼？
不要把真實個資、密碼、別人的祕密貼給 AI，因為內容會離開你的裝置、存在對方伺服器上。
3. AI 給的答案為什麼不能照單全收？該怎麼做？
因為 AI 會「一本正經地說錯」（幻覺），它是猜最像樣的話。應把答案當草稿、參考，重要資訊自己再查證。

點 ChatGPT 程式碼右上角的 Copy code 複製全部程式碼，再貼到編輯器裡，存成一個「英文檔名 + .html」的檔案。最推薦用你前面學過的 VS Code 來存（也可以用記事本）：

  • VS Code（最推薦，前面剛學過）：打開 VS Code → 檔案 → 開新檔案 → 把程式碼貼上 → 檔案 → 另存新檔 → 檔名打 index.html（英文、全小寫）→ 位置選「桌面」→ 存檔。
  • Windows：開「記事本」(Notepad) → 貼上 → 點 檔案 → 另存新檔 → 存檔類型選 所有檔案(*.*) → 檔名打 index.html → 編碼選 UTF-8 → 位置選「桌面」→ 存檔。
  • Mac：開「文字編輯」(TextEdit) → 上方選單 格式 → 製作純文字 → 貼上 → 檔案 → 儲存 → 檔名 index.html → 取消勾選「如果沒有副檔名就用 .txt」→ 存到桌面。
⚠️ 最關鍵的一步：檔名要「英文 + .html」
檔名一定要用英文、而且結尾是 .html——這份作業請固定取名 index.html（全小寫）。GitHub 網站的「首頁」固定只認 index.html 這個名字。
千萬不要用中文檔名（像「我的網頁.html」），也不要讓結尾變成 .txt——中文檔名放到網路上、出現在網址裡很容易出問題，首頁也會打不開。
3

先在自己電腦打開看看（本機預覽）

＝ 對應課程：在「自己電腦」打開，網址會是 file:// ——別人連不到

在桌面的 index.html 上連點兩下，它會用瀏覽器打開。確認畫面、互動按鈕都正常。

  • 看一下瀏覽器網址列：開頭是 file:///…，這代表它只在你這台電腦上，把網址傳給同學，他們是打不開的。
  • 這正是課本第 1～2 章說的：還沒走進網際網路、沒有公有 IP，外面找不到。下一步就來解決它。
4

登入 GitHub，建立一個新的 Repository（倉庫）

＝ 對應課程：準備把網頁搬上「網際網路」這張大網

到 github.com 登入後：

  • 點右上角的 ＋ → New repository（新倉庫）。
  • Repository name 取一個英文名字，例如 my-first-site。
  • 下面選 Public（公開）——要全世界連得到，一定要公開。
  • 勾選 Add a README file，然後點綠色的 Create repository。
5

把桌面的 index.html「拖」進 GitHub 上傳

＝ 對應課程：你的網頁正式「走進網際網路」

在剛建好的倉庫頁面：

  • 點 Add file → Upload files（上傳檔案）。
  • 把桌面的 index.html 直接用滑鼠拖進那塊「Drag files here」的虛線區域（或點 choose your files 選檔）。
  • 等檔名出現後，往下捲，點綠色的 Commit changes（送出變更）。
✅ 檢查點
回到倉庫首頁，要能看到 index.html 出現在檔案清單裡，才算上傳成功。
6

開啟 GitHub Pages，讓它變成真正的網站

＝ 對應課程：拿到公有 IP + 好記網域 + https 上鎖 + 登記過的身分

上傳檔案還只是「存起來」，要再打開 Pages 功能才會變成網站：

  • 在倉庫頁面點上方的 Settings（設定）。
  • 左邊選單往下找到 Pages。
  • Branch 那一欄，把 None 改成 main，資料夾保持 / (root)，按 Save。
7

等 1～2 分鐘，拿到你的公開網址

＝ 對應課程：DNS 把好記名字翻成 IP，全世界都連得到

儲存後，在同一個 Pages 頁面等 1～2 分鐘、重新整理，上方會出現一行綠色的：

Your site is live at https://你的username.github.io/my-first-site/
  • 點 Visit site 就能打開你的網站，把這個網址複製下來，這就是你要繳交的東西。
  • 看網址開頭：是 https://（有鎖頭、有上鎖）、有好記的名字——課本第 3、4 章親眼驗證！
8

用「手機 + 行動網路」打開，驗證它真的在網路上

＝ 對應課程：行動網路 → 基地台 → 網際網路 → DNS → 你的網站

拿手機關掉 Wi-Fi、改用行動網路（4G/5G），打開你的網址。

  • 能打開，就證明它不是只在你電腦的區域網路裡了——它真的走進了網際網路，全世界（包含用基地台的手機）都連得到。
  • 傳給一位同學或家人試試，他們也打得開，恭喜你做出了一個真正的公開網站！
9

（加分）回 ChatGPT 修改，再上傳一次

＝ 體驗「改版→重新部署」的完整流程

想改內容的話，回 ChatGPT 說「幫我把背景改成深色」「多加一個我的作品區」，拿到新程式碼，重存 index.html，再用 Step 5 的方式上傳一次（這次選 覆蓋），稍等一下，網站就會更新了。

網址打開是空白／一片白

通常是網址打錯，或檔案放在子資料夾裡。確認 index.html 在倉庫的最外層（root），而且網址結尾有那串倉庫名稱。
